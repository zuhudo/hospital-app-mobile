# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | ✅ Yes             |
| < 1.0   | ❌ No              |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT** open a public issue
2. Email: **security@zuhudo.com** (or use GitHub's private vulnerability reporting)
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Fix release**: Depends on severity (critical issues within 48 hours)

## Security Best Practices

- Never commit `.env` files or secrets
- Use environment variables for configuration
- Keep dependencies updated
- Run security audits regularly

## Bug Bounty

We currently do not have a bug bounty program, but we deeply appreciate responsible disclosure and will credit reporters in our release notes.
